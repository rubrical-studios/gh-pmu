## Description

`gh pmu branch close` fails on Windows with:

```
fork/exec C:\Program Files\GitHub CLI\gh.exe: The filename or extension is too long
```

The error originates from `GetProjectItemsByIssues()` in `internal/api/queries.go:946`, which passes the entire GraphQL query as a command-line argument via `-f query=<QUERY>`. When a branch has many issues, the query string exceeds Windows' ~32KB command-line limit.

## Root Cause

`queries.go:946` constructs a dynamic GraphQL query with aliases per issue (~800-1000 chars each) and passes it as a CLI argument:

```go
cmd := exec.Command("gh", "api", "graphql", "-f", "query="+query)
```

With 30+ issues in a release, the command easily exceeds Windows limits.

## Affected Code Paths

All locations using `-f query=` pattern with dynamic/unbounded query sizes:

| File | Function | Line | Risk |
|------|----------|------|------|
| `queries.go` | `GetProjectItemsByIssues` | 946 | **HIGH** - Unbounded by issue count |
| `queries.go` | `getProjectFieldsForIssuesBatch` | 1773 | MEDIUM - Has 50-item batch limit |
| `mutations.go` | `GetLabelIDsBatch` | 900 | LOW - Usually small |
| `queries.go` | `GetSubIssueCounts` | ~1349 | MEDIUM |
| `queries.go` | `GetSubIssuesBatch` | ~1425 | MEDIUM |

## Correct Pattern (Already Exists)

`executeBatchMutation()` at `mutations.go:1903` already uses the stdin approach:

```go
cmd := exec.Command("gh", "api", "graphql", "--input", "-")
cmd.Stdin = strings.NewReader(requestBody)
```

## Fix

Refactor all `-f query=` call sites to use `--input -` with stdin, matching the existing `executeBatchMutation()` pattern.
